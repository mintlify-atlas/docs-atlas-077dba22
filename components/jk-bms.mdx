---
title: "jk_bms: ESPHome UART Component for JK-BMS Monitoring"
description: "Monitor and control JK-BMS battery management systems over UART with ESPHome. Full reference for sensors, binary sensors, switches, and text sensors."
---

The `jk_bms` component integrates JK-BMS battery management systems with ESPHome over the **GPS/UART port** using the proprietary JK Modbus protocol. It exposes cell voltages, temperatures, protection thresholds, and operational flags as ESPHome entities, and allows toggling the charge and discharge MOSFETs via switches.

This component acts as a **polling device** on top of the `jk_modbus` bus. You must first define a `jk_modbus` hub that owns the UART interface, then attach one or more `jk_bms` instances to it. Because `MULTI_CONF = True`, multiple BMS units can share the same RS-485 bus with different device addresses.

<Note>
  Requires ESPHome **2025.7.0** or newer. The component auto-loads `jk_modbus`, `binary_sensor`, `sensor`, `switch`, and `text_sensor`.
</Note>

## Hardware Setup

Connect an RS-485-to-TTL adapter (or a direct UART connection from the GPS port) between the BMS and your ESP32. The GPS port outputs at **115200 baud**.

```
JK-BMS GPS port  ──►  RS-485/TTL adapter  ──►  ESP32
```

## Configuration

<Steps>
  <Step title="Define the UART bus">
    Declare a UART component at 115200 baud pointing at the physical pins connected to the adapter.
  </Step>
  <Step title="Define the JK Modbus hub">
    Attach `jk_modbus` to the UART bus and set an appropriate `rx_timeout`.
  </Step>
  <Step title="Declare one or more jk_bms devices">
    Reference the `jk_modbus` hub and set `update_interval`.
  </Step>
  <Step title="Add sensor/switch/text_sensor platforms">
    Each platform block references the `jk_bms` hub via its auto-generated ID.
  </Step>
</Steps>

## Component Configuration

<ParamField path="jk_modbus_id" type="string" required>
  ID of the `jk_modbus` hub this BMS is attached to. Automatically resolved when only one hub is defined.
</ParamField>

<ParamField path="update_interval" type="time" default="5s">
  How often the component polls the BMS for new data. Inherits ESPHome's standard polling component schema. Default is `5s`.
</ParamField>

<ParamField path="id" type="string">
  Optional ID used to reference this BMS instance from sensor/switch/text_sensor platform blocks when multiple BMS units are configured.
</ParamField>

<Note>
  `MULTI_CONF = True` — you can declare multiple `jk_bms` blocks to monitor several battery packs on the same RS-485 bus. Each unit must have a unique Modbus device address (default `0x4E`).
</Note>

### Binary Sensor

| Key | Notes |
|-----|-------|
| `charging` | True while charging MOSFET is active |
| `discharging` | True while discharging MOSFET is active |
| `balancing` | True while cell balancing is active |
| `balancing_switch` | State of the balancing switch setting |
| `charging_switch` | *(Deprecated)* Use `switch.charging` instead |
| `discharging_switch` | *(Deprecated)* Use `switch.discharging` instead |
| `dedicated_charger_switch` | State of the dedicated charger port switch |
| `online_status` | Connectivity — true when BMS is responding |

### Switch

| Key | Register | Notes |
|-----|----------|-------|
| `charging` | `0xAB` | Toggle charge MOSFET on/off |
| `discharging` | `0xAC` | Toggle discharge MOSFET on/off |

<Warning>
  The `balancer` switch (`0x9D`) is defined in source but is commented out because BMS hardware version 11 does not accept writes to that register reliably. Do not expose it unless you have confirmed your hardware supports it.
</Warning>

### Text Sensor

| Key | Notes |
|-----|-------|
| `operation_mode` | Human-readable operation mode string |
| `errors` | Decoded error description string (diagnostic) |
| `battery_type` | Battery chemistry (e.g. LFP, NCM) |
| `password` | BMS password |
| `device_type` | BMS model string (diagnostic) |
| `software_version` | Firmware version string (diagnostic) |
| `manufacturer` | Manufacturer string (diagnostic) |
| `total_runtime_formatted` | Runtime formatted as `0d 0h 0m 0s` |
