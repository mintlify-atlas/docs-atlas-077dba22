All JK-BMS models with software version 6.0 or higher use the implemented protocol and are expected to work. The tables below list every model that community members have confirmed working, together with the hardware version, software version, connection method, and required `protocol_version` setting.

<Note>
  ESPHome 2025.7.0 or higher is required. Software versions below 6.0 use a different protocol and are not supported.
</Note>

## How to identify your protocol version

<Tip>
  Check the hardware version label on your BMS board or in the JikongBMS Android app under device info. Use the following rules:

  - **`JK04`** — hardware version ≤ 3.0 (e.g. JK-B2A16S hw 3.0, sw 3.3.0)
  - **`JK02_24S`** — hardware version ≥ 6.0 and < 11.0
  - **`JK02_32S`** — hardware version ≥ 11.0 (e.g. JK-B2A8S20P hw 11.XW, sw 11.26)

  When in doubt, try `JK02_24S` first for older hardware and `JK02_32S` for newer hardware. The `protocol_version` setting is only required for the BLE component (`jk_bms_ble`). The UART-TTL component (`jk_bms`) detects the protocol automatically.
</Tip>

## NEEY and Heltec Smart Active Balancers

The `heltec_balancer_ble` component supports NEEY and Heltec-branded smart active balancers over BLE. These are separate devices from the BMS and use a different protocol.

| EK-24S15EB (NEEY 15A Smart Active Balancer) | — | — |
| EK-24S10EB (NEEY 10A Smart Active Balancer) | HW-1.1.5 | SW-1.1.1 |
