---
title: 'Monitor a JK-BMS via the RS485 display port (passive)'
description: 'Passively monitor a JK-BMS using its 6-pin display port and an RS485-to-TTL module. Read-only access to cell voltages, temperatures, and pack state.'
---

JK-BMS models from hardware version 10 onwards carry a dedicated **display port** вҖ” a 6-pin JST GH 1.25mm connector that continuously broadcasts pack data over RS485 at 2400 baud. This is the same data stream a Jikong display module would consume. By tapping into it with an RS485-to-TTL adapter and the `jk_bms_display` component, you can monitor cell voltages, temperatures, current, state-of-charge, and protection states without touching the BMS GPS/UART-TTL port or its Bluetooth connection.

<Note>
This connection is **passive and read-only**. The display port protocol is a one-way broadcast: the BMS transmits data continuously and the connected device only listens. Switches and writable number entities are not available through this interface.
</Note>

## Supported models

The display port is present on hardware version вүҘ 10. The connector type varies by model:

| Model | Connector |
|-------|-----------|
| JK-BD6A17S6P / 8P, JK-BD6A20S6P / 8P, JK-BD6A24S6P / 8P | 6-pin Micro JST 1.25mm pitch |
| JK-B1A8S20P, JK-B2A8S20P | 6-pin Micro JST 1.25mm pitch |
| JK-BD6A20S10P, JK-BD6A24S10P | 6-pin Micro JST GH 1.25mm pitch |
| JK-B1A20S15P, JK-B1A24S15P, JK-B2A24S15P, JK-B2A24S20P | 6-pin Micro JST GH 1.25mm pitch |

<Info>
The JK-BD4A17S4P does **not** have a display port. Use the [UART-TTL connection](/connection/uart-ttl) or [BLE connection](/connection/ble) for that model.
</Info>

The **TXD** output of the RS485-to-TTL module connects to the **RX** GPIO of the ESP. No TX wire from the ESP to the module is needed because the interface is receive-only.

| Display port pin | RS485-to-TTL module pin |
|-----------------|------------------------|
| A               | A+                     |
| B               | BвҲ’                     |
| GND             | GND                    |
| VCC             | VCC (5 V)              |
| K+              | вҖ” (do not connect)     |
| KвҲ’              | вҖ” (do not connect)     |

| RS485-to-TTL module pin | ESP pin |
|------------------------|---------|
| RXD                    | GPIO17 (rx\_pin) |
| GND                    | GND    |
| VCC                    | 5 V    |

<Warning>
The K+ and KвҲ’ pins are the power-button interface, not data lines. Shorting K+ to KвҲ’ for one second powers the BMS on; holding for three seconds powers it off. Do not connect these to the RS485 module or the ESP.
</Warning>

## Setup

<Steps>
  <Step title="Install the external component">
    ```yaml
    external_components:
      - source: github://syssi/esphome-jk-bms@main
        refresh: 0s
    ```
  </Step>

  <Step title="Declare the UART bus">
    The display port runs at **2400 baud** вҖ” a much lower speed than the GPS/UART-TTL port. Only the RX pin is needed; there is no TX.

    ```yaml
    substitutions:
      rx_pin: GPIO17

    uart:
      - id: uart_0
        baud_rate: 2400
        rx_pin: ${rx_pin}
    ```
  </Step>

  <Step title="Add the jk_bms_display component">
    ```yaml
    jk_bms_display:
      - id: display0
        uart_id: uart_0
    ```
  </Step>

  <Step title="Declare sensors and binary sensors">
    All available entities for the display port platform are shown below. Remove any you do not need.

    ```yaml
    binary_sensor:
      - platform: jk_bms_display
        system_warning:
          name: "system warning"
        balancing_switch:
          name: "balancing switch"
        charging:
          name: "charging"
        discharging:
          name: "discharging"
        cell_voltage_undervoltage_protection:
          name: "cell voltage undervoltage protection"
        cell_voltage_overvoltage_protection:
          name: "cell voltage overvoltage protection"
        overcurrent_protection:
          name: "overcurrent protection"
        mosfet_overtemperature_protection:
          name: "mosfet overtemperature protection"
        battery_temperature_protection:
          name: "battery temperature protection"
        short_circuit_protection:
          name: "short circuit protection"
        coprocessor_communication_error:
          name: "coprocessor communication error"
        balancer_wire_resistance_too_high:
          name: "balancer wire resistance too high"
        cell_count_mismatch:
          name: "cell count mismatch"

    sensor:

        total_voltage:
          name: "total voltage"
        current:
          name: "current"
        power:
          name: "power"
        charging_power:
          name: "charging power"
        discharging_power:
          name: "discharging power"
        min_cell_voltage:
          name: "min cell voltage"
        max_cell_voltage:
          name: "max cell voltage"
        min_voltage_cell:
          name: "min voltage cell"
        max_voltage_cell:
          name: "max voltage cell"
        state_of_charge:
          name: "state of charge"
        delta_cell_voltage:
          name: "delta cell voltage"
        mosfet_temperature:
          name: "mosfet temperature"
        battery_temperature:
          name: "battery temperature"

          name: "average cell voltage"

    ```
  </Step>

  <Step title="Flash and verify">
    After flashing, open the ESPHome logs. You should see incoming frame data within a few seconds as the BMS broadcasts its next update cycle:

    ```text
    [sensor:127]: 'jk-bms-display total voltage': Sending state 53.64000 V with 2 decimals of accuracy
    [sensor:127]: 'jk-bms-display state of charge': Sending state 99.00000 % with 0 decimals of accuracy
    ```

    If no data appears, verify that the A and B wires are not swapped on the RS485 module вҖ” polarity inversion is the most common wiring mistake with differential RS485.
  </Step>
</Steps>
